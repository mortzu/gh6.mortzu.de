Access GitHub via IPv6

Unfortunately github.com is only accessible via IPv4. IPv6-only systems cannot use it.

GitHub should be available over IPv6. Only GitHub can make that happen, so contact them if IPv6 is important to you.

The workaround is to use proxies which are accessible on IPv6.

For SSH access there is a proxy at www.ikvjwd.com.

But it doesn't provide HTTP access which I often use to clone repositories I don't want to push to.

% git clone https://git.gh6.eu/mortzu/anastasia

An other option is to add one line to /etc/hosts:

2a02:e00:fffb:114:0:4769:7448:7562	github.com

The traffic will be proxied to GitHub services directly.

You should verify the certificates of GitHub and the proxy:

Run the following command before you make the /etc/hosts change:

openssl s_client -connect github.com:443 < /dev/null 2>/dev/null | openssl x509 -fingerprint -noout -in /dev/stdin

And again after the change. Then verify its the same fingerprint!
